# סיווג קשתות ב-DFS

## 🌸 אינטואיציה

כשמריצים DFS על גרף, כל קשת נופלת לאחת מארבע קטגוריות. הסיווג מבוסס על זמני **גילוי** (`d[v]` — מתי נכנסנו לצומת) ו-**סיום** (`f[v]` — מתי יצאנו מהצומת, אחרי שסיימנו את כל הבנים).

הטווחים `[d[v], f[v]]` מתנהגים כמו **סוגריים** — או שתי צמתים זרים לחלוטין, או שאחד מכיל את השני. זו התמונה המרכזית לזכור.

## 📐 ארבעת סוגי הקשתות

עבור קשת `(u, v)` שנבדקת כשנמצאים ב-`u`:

**Tree edge (קשת-עץ):** `v` התגלה על-ידי `u` ב-DFS (כלומר `v` היה לבן כש-`(u,v)` נסרקה). `d[u] < d[v] < f[v] < f[u]`. קיימות בכל גרף (מכוון ולא מכוון).

**Back edge (קשת-אחורה):** `v` הוא אב-קדמון של `u` בעץ ה-DFS. `d[v] < d[u] < f[u] < f[v]` — כלומר `v` עדיין "פתוח" (אפור) כש-`(u,v)` נסרקת. **מעידה על מעגל.**

**Forward edge (קשת-קדימה):** `v` הוא צאצא של `u` אבל לא ישיר (יש מסלול עץ אחר מ-`u` אליו). `d[u] < d[v] < f[v] < f[u]`, ו-`v` כבר שחור כש-`(u,v)` נסרקת. **קיימות רק בגרפים מכוונים.**

**Cross edge (קשת-צולבת):** `u` ו-`v` בתתי-עצים שונים, אף אחד מהם לא אב-קדמון של השני. `d[v] < f[v] < d[u] < f[u]` — `v` סיים לפני ש-`u` התחיל. **קיימות רק בגרפים מכוונים.**

## 📐 פסאודו-קוד

```pseudocode
classify(u, v):
    if color[v] == WHITE:
        return TREE
    if color[v] == GRAY:
        return BACK
    # color[v] == BLACK
    if d[u] < d[v]:
        return FORWARD
    else:
        return CROSS
```

## ⏱️ סיבוכיות

- **זמן:** `O(|V| + |E|)` (חלק מ-DFS רגיל).
- **זיכרון:** `O(|V|)` לצבעים, זמני גילוי/סיום, וערימת הרקורסיה.

## 💡 טיפ למבחן

**בגרף לא מכוון יש רק Tree ו-Back.** זה הכלל החשוב ביותר. אין Forward ואין Cross — כל קשת שפוגשת צומת אפור היא Back, וכל קשת שמגלה צומת לבן היא Tree.

**איתור מעגלים:** גרף מכוון מכיל מעגל **אם ורק אם** ה-DFS מוצא Back edge. בגרף לא מכוון: Back edge על קשת שאינה האב מעידה על מעגל.

**סדר טופולוגי:** אם הגרף מכוון וללא מעגלים (DAG), אין Back edges, והסידור לפי `f[v]` בסדר יורד נותן סדר טופולוגי.

**טעות קלאסית:** לסווג Forward כ-Back. מבדילים לפי צבע של `v` בזמן הסריקה — אפור = Back, שחור + `d[u] < d[v]` = Forward.
