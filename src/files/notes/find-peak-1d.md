# מציאת שיא (Find-Peak)

## 🌸 אינטואיציה

בהינתן מערך `A[1..n]` **לא ממוין**, רוצים למצוא אינדקס `i` כך ש-`A[i] ≥ A[i-1]` וגם `A[i] ≥ A[i+1]` (בקצוות מתעלמים מהשכן שאינו קיים). שימי לב — זה **שיא מקומי**, לא גלובלי. הנקודה היפה: תמיד יש שיא כזה במערך לא ריק.

פורמלית (מהסילבוס): `peak(i) = A[i]` הוא שיא אם `i=1`, או `i>1` ו-`A[i] ≥ A[i-1]`, או במקרה הכללי שגם `A[i] ≥ A[i+1]`.

למה אפשר `O(log n)` במערך לא ממוין? זה נראה מפתיע, אבל הטיעון פשוט: אם האמצע `mid` לא שיא, אז אחד השכנים שלו גדול ממנו. נלך לצד של השכן הגדול — מובטח שקיים שיא בחצי הזה (כי המערך "עולה" לתוכו, והקצה של החצי הזה לא יכול לרדת לאינסוף).

## 📐 פסאודו-קוד

**גרסה לינארית (מהסילבוס) — `O(n)`:**

```pseudocode
find-peak(A, k):
    if k = 1 then return A[k]
    if A[k] >= A[k-1] then return A[k]
    return find-peak(A, k-1)

# first call: find-peak(A, n)
```

הגרסה הרקורסיבית הזו גוררת `T(n) = T(n-1) + O(1) = O(n)`.

**גרסה בינארית (המומלצת) — `O(log n)`:**

```pseudocode
find_peak_bin(A, lo, hi):
    if lo == hi:
        return A[lo]
    mid = (lo + hi) / 2
    if A[mid] < A[mid + 1]:
        return find_peak_bin(A, mid + 1, hi)
    else:
        return find_peak_bin(A, lo, mid)

# first call: find_peak_bin(A, 1, n)
```

## ⏱️ סיבוכיות

- **גרסה לינארית:** זמן `T(n) = T(n-1) + O(1) = O(n)`, זיכרון `O(n)` לרקורסיה (`O(1)` איטרטיבי).
- **גרסה בינארית:** זמן `T(n) = T(n/2) + O(1) = O(log n)`, זיכרון `O(log n)` לרקורסיה (`O(1)` איטרטיבי).

## 💡 טיפ למבחן

**טעות קלאסית:** לחשוב שצריכים למצוא **שיא גלובלי**. הבעיה מבקשת רק שיא מקומי — לכן אפשר חיפוש בינארי, מה שאי-אפשר למקסימום גלובלי במערך לא ממוין.

**הוכחת נכונות הבינארי (טיעון שכיח במבחן):** אם `A[mid] < A[mid+1]`, אז בחצי הימני `[mid+1, hi]` קיים שיא — נגדיר `x = A[mid+1]`, הוא גדול משכן שמאלי בתוך החצי. אם לא הוא שיא, שכן ימני גדול ממנו, וכך מתקדמים — עד שנתקעים בשיא או בקצה הימני (שתמיד שיא בגלל המונוטוניות). אותו טיעון סימטרי לחצי השמאלי.

**איך מזהים את זה במבחן?** כל פעם שהשאלה היא "מצא אלמנט שגדול משכניו" או "מצא מקום ש-`A[i] ≥ A[i±1]`" — זה Find-Peak, ופתרון `O(log n)` צפוי.

**הרחבה דו-ממדית:** אפשר להרחיב למטריצה `n×m` בזמן `O(n log m)` (או `O(m log n)`) עם רעיון דומה על עמודות — בוחרים את העמודה האמצעית, מוצאים בה את המקסימום, ומשווים לשני השכנים האופקיים.
