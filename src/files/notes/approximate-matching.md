# התאמה מעגלית / DNA בקירוב

## 🌸 אינטואיציה

שתי בעיות שדורשות "גמישות" מעבר להתאמה מדויקת: סיבובים מעגליים והתאמות עם טעויות.

**התאמה מעגלית:** בהינתן שתי מחרוזות `T` ו-`T'` באותו אורך, לבדוק אם `T'` הוא **סיבוב מעגלי** של `T`. למשל, `T = "ABCDE"` ו-`T' = "CDEAB"` — כן. `T = "ABCDE"` ו-`T' = "ACBDE"` — לא.

**הטריק היפה:** אם ניקח `T + T` (שרשור של `T` עם עצמו), אז כל סיבוב מעגלי של `T` הוא **תת-מחרוזת** של `T + T`. לכן הבעיה מצטמצמת לחיפוש `T'` בתוך `T + T` — וזה KMP/Rabin-Karp סטנדרטי.

**DNA בקירוב (Approximate Matching):** נתונות מחרוזות `T` (טקסט) ו-`P` (תבנית), וסף `k`. רוצים למצוא את כל ההופעות של `P` ב-`T` עם **לכל היותר `k` טעויות** (החלפה / הוספה / מחיקה של תו). קריטי בביולוגיה חישובית (התאמת רצפי DNA עם מוטציות).

## 📐 פסאודו-קוד

**התאמה מעגלית:**

```pseudocode
is_rotation(T, T_prime):
    if |T| != |T_prime|:
        return FALSE
    return KMP_search(T + T, T_prime)   # is T_prime a substring?
```

**DNA בקירוב (פתרון נאיבי מבוסס Edit Distance):**

```pseudocode
approximate_match(T, P, k):
    n = |T|; m = |P|
    results = []
    for i = 1 to n - m + 1:
        # check window T[i..i+m-1] against P
        d = edit_distance(T[i..i+m-1], P)
        if d <= k:
            results.append(i)
    return results
```

## ⏱️ סיבוכיות

- **התאמה מעגלית:** `O(|T| + |T'|)` עם KMP.
- **DNA בקירוב — נאיבי:** `O(|T| · |P|²)` (חישוב edit distance לכל חלון), או `O(|T| · |P| · k)` עם גרסה מצומצמת ל-`k` טעויות.
- **שיטות מתקדמות:** Bit-parallel Shift-And יורד ל-`O(|T| · ⌈m/w⌉)` היכן ש-`w` הוא גודל מילת מחשב.

## 💡 טיפ למבחן

**"סיבוב מעגלי" = מיד לחשוב על `T + T`.** זה אחד הטריקים הכי מזוהים באלגוריתמיקה. אם במבחן שואלים על שרשראות מפתחות עגולות, מחרוזות סיבוב, או "cyclic rotation" — השרשור הכפול הוא הפתרון.

**DNA בקירוב — הבחנה חשובה:**
- אם מותרות רק **החלפות** (Hamming distance): קל יותר, אפשר `O(|T| · |P|)`.
- אם מותרות **הוספות ומחיקות** (Edit / Levenshtein distance): מסובך יותר, דורש DP.

**הקשר ל-Edit Distance:** החלק המרכזי בגרסה הכללית הוא חישוב מרחק עריכה — טבלת DP בגודל `(|P|+1) × (|T|+1)`.

**טעות קלאסית:** לחשוב ש-`T + T` מספיק גם אם `|T'| > |T|`. חובה לבדוק קודם שהאורכים שווים — אחרת אפילו אם `T'` נמצא ב-`T + T`, זה לא סיבוב.
