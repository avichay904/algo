# בדיקת גרף דו-צדדי (Bipartite Check)

## 🌸 אינטואיציה

גרף **דו-צדדי** הוא גרף שבו אפשר לחלק את הצמתים לשתי קבוצות `U` ו-`V` כך שכל קשת מחברת צומת ב-`U` עם צומת ב-`V` (אין קשתות בתוך אותה קבוצה). שקול לומר שאפשר לצבוע את הצמתים בשני צבעים כך שלא תהיינה שתי צמתים שכנים באותו צבע.

**טענה חשובה:** גרף דו-צדדי אם ורק אם **אין בו מעגל באורך אי-זוגי**.

איך בודקים? עושים BFS מצומת התחלה, צובעים אותו ב"אדום", וכל שכבה הבאה צובעים בצבע ההפוך. אם מוצאים קשת בין שני צמתים באותה שכבה (או יותר כללי — שני צמתים בצבע זהה) — יש מעגל אי-זוגי, והגרף **אינו** דו-צדדי.

## 📐 פסאודו-קוד

```pseudocode
is_bipartite(G, s):
    color[v] = NONE for all v
    color[s] = RED
    Q = queue with s

    while Q not empty:
        u = Q.dequeue()
        for each neighbor v of u:
            if color[v] == NONE:
                color[v] = opposite(color[u])
                Q.enqueue(v)
            else if color[v] == color[u]:
                return FALSE   # found an odd cycle
    return TRUE
```

**הערה:** אם הגרף לא קשיר — יש להפעיל את הלולאה הזו לכל רכיב קשירות.

## ⏱️ סיבוכיות

- **זמן:** `O(|V| + |E|)` (BFS רגיל).
- **זיכרון:** `O(|V|)` לצבעים ולתור.

## 💡 טיפ למבחן

**מעגל אי-זוגי = לא דו-צדדי** — זה המשפט המרכזי לזכור. אם בשאלת מבחן מופיע גרף עם מעגל באורך 3 (משולש), מיידית אפשר לומר שהוא לא דו-צדדי.

**שימושים:** התאמות (matching) דו-צדדיות — כל אלגוריתם התאמה דו-צדדי (Hopcroft–Karp וכד') דורש קודם לוודא שהגרף אכן דו-צדדי.

**גרסה עם DFS:** אפשר לעשות גם עם DFS במקום BFS — העיקרון זהה, צובעים לסירוגין לפי עומק.

**גרף לא קשיר:** אל תשכחי — אם הגרף לא קשיר, צריך להפעיל את הבדיקה לכל רכיב בנפרד. גרף דו-צדדי = **כל** הרכיבים שלו דו-צדדיים.
