# החלפת מטבעות — גישה חמדנית

## 🌸 אינטואיציה

בעיית החלפת המטבעות: נתונה קבוצת ערכי מטבעות `C = {c_1, c_2, ..., c_k}` וסכום יעד `S`. רוצים לייצג את `S` כסכום מטבעות מ-`C` (אפשר לחזור על מטבע), במינימום מטבעות.

**הגישה החמדנית:** בוחרים כל פעם את המטבע הגדול ביותר שלא עובר את מה שנשאר. פשוט, מהיר ו-`O(k log k + k)`.

**אבל לא תמיד נכון!** הגישה החמדנית עובדת רק במערכות מטבעות "קנוניות" — כמו המערכת האמריקאית `{1, 5, 10, 25}`, או `{1, 2, 5, 10, 20, 50, 100}`. לקבוצות מטבעות שרירותיות צריך **תכנון דינמי**.

## 📐 פסאודו-קוד

```pseudocode
greedy_coin_change(C, S):
    # C is sorted in decreasing order
    result = []
    for c in C:                  # from largest to smallest
        while S >= c:
            result.append(c)
            S -= c
    if S == 0:
        return result
    else:
        return FAILURE
```

## ⏱️ סיבוכיות

- **זמן:** `O(k log k)` למיון + `O(S / c_min)` לבחירה, בדרך כלל `O(k + S/c_min)`.
- **זיכרון:** `O(S/c_min)` לפלט.

## 💡 טיפ למבחן

**דוגמה נגדית קלאסית:** `C = {1, 3, 4}`, יעד `S = 6`.
- **חמדני:** לוקח `4`, נשאר `2`, לוקח `1+1` → 3 מטבעות.
- **אופטימום:** `3 + 3` → 2 מטבעות.

אז כאן הגישה החמדנית **נכשלת**. למערכת הזו צריך DP.

**איך מוכיחים שמערכת קנונית?** נדרשות **טענת החלפה (exchange argument)** ואינדוקציה. בגרסה פורמלית, מראים שלכל פתרון אופטימלי, אפשר להחליף מטבעות קטנים במטבע הגדול ביותר שעובר את הסכום, בלי להגדיל את מספר המטבעות.

**איך מזהים במבחן?** כששואלים "האם הגישה החמדנית עובדת?" — תמיד לחפש דוגמה נגדית. אם יש מטבעות **לא** בחזקות של ערכים קבועים (כמו 1, 3, 4) זה חשוד.

**פתרון DP (לא חמדני):** `coins[S] = 1 + min_{c in C, c <= S} coins[S - c]`, `coins[0] = 0`. זמן `O(S · k)`.
