# רקורסיה ומשפט האב

## 🌸 אינטואיציה

הרבה אלגוריתמים "מחלק וכבוש" מחלקים קלט בגודל `n` ל-`a` תתי-בעיות בגודל `n/b`, ואז עושים עבודה של `f(n)` כדי לאחד את התוצאות. משפט האב נותן לנו **תשובה מיידית** לפתרון יחס הנסיגה `T(n) = a·T(n/b) + f(n)` בלי לפתוח רקורסיות עד הסוף.

הרעיון: משווים את העבודה בכל רמה. יש עץ רקורסיה עם גובה `log_b n`. בכל רמה יש יותר תתי-בעיות, אבל כל אחת קטנה יותר. השאלה היא איזה חלק דומיננטי — השורשים או העלים? משפט האב עונה על זה דרך השוואה בין `log_b a` (קצב הגדילה של מספר תתי-הבעיות) לבין `d` (הקצב שבו `f(n) = Θ(n^d)` גדל).

## 📐 פסאודו-קוד

```pseudocode
master_theorem(a, b, d):
    # T(n) = a * T(n/b) + Θ(n^d), with a >= 1, b > 1, d >= 0
    c = log_b(a)
    if d > c:
        return Θ(n^d)           # Case 3: root dominates
    if d == c:
        return Θ(n^d * log n)   # Case 2: balanced
    if d < c:
        return Θ(n^c)           # Case 1: leaves dominate
```

## ⏱️ סיבוכיות

- **זמן:** תלוי במקרה — `Θ(n^d)`, `Θ(n^d log n)`, או `Θ(n^{log_b a})`.
- **זיכרון:** לרוב `O(log n)` לערימת הרקורסיה (תלוי במימוש).

## 💡 טיפ למבחן

**דוגמת עבודה (מהסילבוס):** `T(n) = 2T(n/2) + 7n`. פה `a=2, b=2, f(n)=7n → d=1`. מחשבים `log_b a = log_2 2 = 1 = d`, אז זה **מקרה מאוזן**: `T(n) = Θ(n^d · log n) = Θ(n log n)`. (זה בדיוק Merge Sort!)

**דוגמאות נוספות לתרגול:**
- `T(n) = 8T(n/2) + n²` → `a=8, b=2, d=2`. `log_2 8 = 3 > 2 = d` → **העלים שולטים**: `Θ(n^3)`.
- `T(n) = 2T(n/2) + n²` → `a=2, b=2, d=2`. `log_2 2 = 1 < 2 = d` → **השורש שולט**: `Θ(n²)`.
- `T(n) = 4T(n/2) + n²` → `a=4, b=2, d=2`. `log_2 4 = 2 = d` → **מאוזן**: `Θ(n² log n)`.

**טעות קלאסית:** לא לשכוח ש-`f(n)` חייב להיות פולינומי (`Θ(n^d)`). אם יש לוגריתמים, למשל `f(n) = n log n`, משפט האב הבסיסי לא חל — צריך את הגרסה המורחבת (Akra–Bazzi) או פתיחת עץ רקורסיה.

**זיהוי מהיר במבחן:** כשרואים `T(n) = a·T(n/b) + n^d`, חשבי מיד `log_b a` והשווי ל-`d`. זה חוסך דקות יקרות.
