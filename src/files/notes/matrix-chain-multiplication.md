# כפל שרשרת מטריצות (Matrix Chain Multiplication)

## 🌸 אינטואיציה

נתונות n מטריצות `A₁, A₂, ..., Aₙ` כך שניתן לכפול אותן ברצף. מכיוון שכפל מטריצות הוא **אסוציאטיבי** אבל **לא קומוטטיבי**, סדר הכפל משפיע על מספר פעולות הכפל הסקלרי.

**דוגמה**: עבור `A: 50×2, B: 2×100, C: 100×10`:
- `(A·B)·C` = `50·2·100 + 50·100·10` = 10,000 + 50,000 = **60,000**
- `A·(B·C)` = `2·100·10 + 50·2·10` = 2,000 + 1,000 = **3,000**

השיפור פי 20! אנחנו מחפשים את **החלוקה הסוגריים האופטימלית**.

## 📐 פסאודו-קוד

```
MATRIX-CHAIN-ORDER(p):
  # p[0..n] הוא מערך המימדים, כך ש-Aᵢ היא p[i-1] × p[i]
  n = length(p) - 1
  m = table[1..n][1..n]       # m[i][j] = מספר פעולות מינימלי לכפל Aᵢ..Aⱼ
  s = table[1..n][1..n]       # s[i][j] = נקודת החלוקה האופטימלית

  for i = 1 to n:
    m[i][i] = 0

  for L = 2 to n:              # L = אורך השרשרת
    for i = 1 to n-L+1:
      j = i+L-1
      m[i][j] = ∞
      for k = i to j-1:
        q = m[i][k] + m[k+1][j] + p[i-1]·p[k]·p[j]
        if q < m[i][j]:
          m[i][j] = q
          s[i][j] = k

  return m, s
```

שחזור הפתרון:
```
PRINT-OPTIMAL-PARENS(s, i, j):
  if i == j: print "A" + i
  else:
    print "("
    PRINT-OPTIMAL-PARENS(s, i, s[i][j])
    PRINT-OPTIMAL-PARENS(s, s[i][j]+1, j)
    print ")"
```

## ⏱️ סיבוכיות

- **זמן:** `Θ(n³)` — שלושה לולאות מקוננות
- **זיכרון:** `Θ(n²)` — טבלאות m ו-s

## 💡 טיפ למבחן

- הכי קריטי: להבין ש-**`m[i][j] = min_{i≤k<j} (m[i][k] + m[k+1][j] + p[i-1]·p[k]·p[j])`**
- הוספת p[i-1]·p[k]·p[j] היא **עלות הכפל של שתי התוצאות** ששרשרתנו — מימדים: (p[i-1] × p[k]) · (p[k] × p[j]) = p[i-1] × p[j]
- החישוב **מבצע לולאות לפי אורך תת-בעיה** (L), לא לפי i ו-j ישירות — אחרת תיצרי תת-בעיות שאת לא מכירה את תוצאתן
- לא לבלבל עם Optimal BST — נוסחאות דומות אבל לא זהות
