# חיתוך מוט / ספירת מסלולי לוח

שתי בעיות קלאסיות של תכנון דינמי שנראות דומות במבט ראשון, אבל עם ניואנסים שונים.

## 🌸 אינטואיציה

**חיתוך מוט:** מוט באורך `n`. נתונים מחירים `p[1..n]` — המחיר של חתיכה באורך `i` הוא `p[i]`. איך לחתוך את המוט למקסימום הכנסה?
הרעיון: החתך הראשון יכול להיות באורך `i` לכל `1 ≤ i ≤ n`, ואז נשאר לחתוך באופן אופטימלי מוט באורך `n - i`.

**מסלולי לוח (בניסוח הסילבוס):** לוח `n × m`. מתחילים בתא `(2, m)` (או יותר כללי — בשורה `i ≥ 2`) ורוצים להגיע ל-`(1, 1)`. מכל תא `(i, j)` אפשר לצעוד לכל תא בשורה `i-1` שנמצא משמאל — `(i-1, 1), (i-1, 2), ..., (i-1, j-1)`. כמה מסלולים יש?
הרעיון: `T(i, j) = Σ_{k=1}^{j-1} T(i-1, k)`, עם `T(1, j) = 1` (מגיעים בצעד אחד).

(הערה: במרבית הספרים מופיע ניסוח "ימינה/למטה" — `paths(i,j) = paths(i-1,j) + paths(i,j-1)` — זו וריאציה שכיחה אך שונה. פירטנו כאן את הגרסה של הקורס.)

## 📐 פסאודו-קוד

**חיתוך מוט:**

```pseudocode
rod_cut(p, n):
    r[0] = 0
    for j = 1 to n:
        best = -infinity
        for i = 1 to j:
            best = max(best, p[i] + r[j - i])
        r[j] = best
    return r[n]
```

**מסלולי לוח (בניסוח הסילבוס):**

```pseudocode
count_paths(i, j):
    if i == 1:
        return 1
    total = 0
    for k = 1 to j - 1:
        total += count_paths(i - 1, k)
    return total

# DP iterative (bottom-up):
count_paths_DP(n, m):
    for j = 1 to m: T[1][j] = 1
    for i = 2 to n:
        for j = 1 to m:
            T[i][j] = sum( T[i-1][k] for k = 1..j-1 )
    return T[n][m]
```

## ⏱️ סיבוכיות

- **חיתוך מוט:** זמן `O(n²)`, זיכרון `O(n)`.
- **מסלולי לוח (ניסוח הסילבוס):** זמן `O(n·m²)` במימוש הישיר, או `O(n·m)` כשמצברים prefix-sum בכל שורה. זיכרון `O(n·m)` (ניתן לצמצם לשתי שורות).

## 💡 טיפ למבחן

**דוגמה למסלולי לוח (מהסילבוס):** על לוח `2 × 7`, מספר המסלולים מ-`(2,7)` ל-`(1,1)` הוא `T(2,7) = T(1,6) + T(1,5) + T(1,4) + T(1,3) + T(1,2) + T(1,1) = 6`. **זכרי את הניסוח המדויק:** זה לא הסידור הקומבינטורי `C(n+m-2, n-1)` המוכר, אלא הסיכום שלעיל.

**חיתוך מוט — טעות קלאסית:** לשכוח את האפשרות **לא לחתוך בכלל**. הנוסחה `r[j] = max(p[j], max_{i<j}(p[i] + r[j-i]))` סופרת את זה בעצמה כי `i` יכול להיות `j`, ואז `r[j-i] = r[0] = 0`.

**דוגמה לחיתוך מוט:** `p = [1, 5, 8, 9]`, `n = 4`. `r[1]=1, r[2]=max(5, 1+1)=5, r[3]=max(8, 1+5, 5+1)=8, r[4]=max(9, 1+8, 5+5, 8+1)=10` — התשובה היא 10 (שני חתכים באורך 2).

**תת-מבנה אופטימלי:** שתי הבעיות הן דוגמה מופתית לתת-מבנה אופטימלי — פתרון אופטימלי של הבעיה הגדולה מורכב מפתרונות אופטימליים של בעיות קטנות יותר. הוכחת האופטימליות בחיתוך מוט היא באמצעות טיעון החלפה: אם בחתך המינימלי לא היה `p[i] + r[n-i]`, אפשר להחליף ולקבל פתרון שלא גרוע יותר.
