# בעיית הקפיצה (Jump DP)

## 🌸 אינטואיציה

יש מערך `A[1..n]` עם עלויות. מתחילים בתא 1 ורוצים להגיע לתא `n`. בכל צעד אפשר ללכת לתא הבא (`i → i+1`) או לדלג על תא (`i → i+2`). על פי הניסוח של הקורס, **העלות של דילוג לתא `i` היא `A[i]`** — כלומר משלמים רק כשמדלגים (קפיצה של 2), ואילו על צעד רגיל (קפיצה של 1) לא משלמים. רוצים את המסלול הזול ביותר.

הרעיון: אם אני בתא `n`, הגעתי אליו מ-`n-1` (בצעד, חינם) או מ-`n-2` (בדילוג, בעלות `A[n]`). לכן העלות המינימלית ל-`n` היא המינימום בין שתי האפשרויות. זה תת-מבנה אופטימלי קלאסי — **תכנון דינמי**.

**יחס הנסיגה (מהסילבוס):**
- `T(0) = 0`
- `T(1) = A[1]`
- `T(n) = min(T(n-1), T(n-2) + A[n])` עבור `n > 1`

## 📐 פסאודו-קוד

גרסה רקורסיבית (מתוך הסילבוס):

```pseudocode
jump(a[], n):
    if n < 2 then return a[n]
    return min( jump(a[], n-1), jump(a[], n-2) + a[n] )
```

הרקורסיה הזו נופלת לסיבוכיות אקספוננציאלית (`Θ(2^(n/2))`), לכן מוסיפים זיכרון (DP Bottom-Up):

```pseudocode
jump_DP(a, n):
    Jump[0] = a[0]
    Jump[1] = a[1]
    for i = 2 to n:
        Jump[i] = min( Jump[i-1], Jump[i-2] + a[i] )
    return Jump[n]
```

ואפשר עוד לצמצם זיכרון ל-`O(1)` בעזרת שני משתנים בלבד (rolling).

## ⏱️ סיבוכיות

- **רקורסיה נאיבית:** זמן `Θ(2^(n/2))`, זיכרון `O(n)` לערימה.
- **DP איטרטיבי:** זמן `O(n)`, זיכרון `O(n)` לטבלת `Jump`, ניתן לצמצם ל-`O(1)`.

## 💡 טיפ למבחן

**החלק הקריטי: מי משלם?** בניסוח הקורס, **רק הדילוג (קפיצה של 2) עולה `A[n]`** — הצעד הרגיל חינם. אל תתבלבלי עם ניסוחים אחרים שמוסיפים `A[i]` תמיד. הוכחה באינדוקציה במבחן: הניחי שהרקורסיה נכונה עד `n-1`, ואז פיצול ל-`T(n-1)` ול-`T(n-2)+A[n]` מכסה את שתי האפשרויות האחרונות בדיוק, לכן המינימום שלהן נותן את `T*(n)`.

**תנאי הבסיס** הם הכי חשובים — בניסוח הקורס `Jump[0]=a[0]` ו-`Jump[1]=a[1]`. אל תחליפי ל-`0` — זה משבש את הצעד הראשון.

**איך מזהים Jump DP?** הבעיה מתארת סדרה של החלטות **מקומיות** (לדלג או לא) שכל אחת משפיעה רק על התא הבא — בדיוק המבנה של DP חד-ממדי עם יחס נסיגה של שני מונחים.

**הכללה:** אם מותרות קפיצות עד `k` תאים קדימה, הנוסחה הופכת ל-`Jump[i] = min_{j=1..k} (Jump[i-j] + cost(j, i))`, עדיין `O(n·k)`.
