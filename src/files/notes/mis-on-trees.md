# קבוצה בלתי תלויה מרבית בעץ (Maximum Independent Set on Trees)

## 🌸 אינטואיציה

**קבוצה בלתי תלויה** בגרף היא קבוצת צמתים שאין ביניהם קשתות. רוצים למצוא קבוצה כזו עם סכום משקלים מקסימלי. בגרף כללי זו בעיה NP-קשה, אבל **בעץ** יש פתרון אלגנטי ב-`O(n)` עם DP.

הרעיון: לכל צומת `v` נגדיר שני ערכים:
- `I(v)` = ערך ה-MIS בתת-העץ של `v` כאשר **חייבים לכלול** את `v`.
- `E(v)` = ערך ה-MIS בתת-העץ של `v` כאשר **לא כוללים** את `v`.

אם כוללים את `v`, אסור לכלול את בניו. אם לא כוללים את `v`, כל בן יכול לבחור לעצמו אם להיכלל או לא — ניקח את המקסימום לכל בן באופן עצמאי. התשובה הסופית (עבור השורש `r`) היא `max(I(r), E(r))`.

## 📐 פסאודו-קוד

```pseudocode
# F[v] = weight of node v
# children(v) = list of child nodes in the rooted tree

MIS(v):
    if v is a leaf:
        I[v] = F[v]
        E[v] = 0
        return

    for u in children(v):
        MIS(u)

    I[v] = F[v] + sum_{u in children(v)} E[u]
    E[v] = sum_{u in children(v)} max(I[u], E[u])

# answer = max(I[root], E[root])
```

## ⏱️ סיבוכיות

- **זמן:** `O(n)` — כל צומת נבדקת פעם אחת.
- **זיכרון:** `O(n)` לטבלאות `I`, `E` ולערימת הרקורסיה.

## 💡 טיפ למבחן

**איך בוחרים שורש?** אפשר לבחור כל צומת בתור השורש. הבחירה לא משפיעה על התשובה — זה עץ, אז הקבוצות הבלתי תלויות זהות ללא תלות בכיוון.

**דוגמה לתרגול (מהסילבוס):** עץ עם משקלים שהסכום האופטימלי שלהם הוא `20 = 2 + 4 + 5 + 2 + 3 + 4` — אלו הצמתים שנבחרים ב-MIS (סכום משקלים מקסימלי כך ששני צמתים שכנים לא נבחרים יחד).

**טעות קלאסית:** לשכוח את תנאי הבסיס (עלה: `I[v]=F[v], E[v]=0`). טעות נוספת: כשמחשבים `E[v]` חושבים שחייבים להוציא את כל הבנים — לא, פשוט כל בן עומד בזכות עצמו ולוקחים `max(I[u], E[u])`.

**הכללה:** הרעיון הזה (DP על עץ עם שני מצבים לצומת) עובד לעוד הרבה בעיות: Vertex Cover בעץ, Dominating Set בעץ, ועוד.
